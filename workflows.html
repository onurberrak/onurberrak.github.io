// implementation · n8n + claude templates

The wiring, opened up.

For each workflow above: the n8n node sequence and the Claude prompts that do the heavy lifting. Drop these into n8n, swap in your API keys and ICP rules, and the loop is yours. Production-tested patterns, not hand-wavy pseudo-code.

01 · Outbound machine n8n + Claude

// n8n nodes

1. Cron (Schedule Trigger)            · daily 06:00
2. HTTP Request → Apollo            · POST /v1/mixed_people/search
                                       body: ICP filters (titles, employees, region, signals)
3. Function (n8n)                   · dedupe vs. existing HubSpot contacts
4. HTTP Request → Anthropic         · POST /v1/messages, claude-sonnet-4-7
                                       prompt: fit-score (see below)
5. IF node                          · score >= 7 ? branch
   ├─ 6a. HTTP → Instantly          · POST /api/v1/lead/add
   │                                  campaign: "EU outbound · sales leaders"
   └─ 6b. HTTP → HubSpot            · contact create with tag "nurture-low-fit"
7. Webhook (Instantly reply)        · listen on /webhook/reply
8. HTTP → Anthropic                 · classify reply (interested / not now / etc.)
9. HTTP → HubSpot                   · create deal
10. Slack → channel #sales-warm     · DM the KAM with full context

// claude · fit scoring prompt

You are scoring a B2B sales lead against this ICP:
- Companies: 50–500 employees, B2B SaaS or business services, DACH or EU
- Buyer titles: Head of Sales, VP Sales, CRO, Founder, RevOps lead
- Signals: hiring sales reps, recent funding, expansion into DACH

LEAD:
{ name, title, company, headcount, industry, region, signals }

Return JSON only:
{
  "score": <1-10>,
  "reason": "<one sentence>",
  "top_question": "<one discovery question tailored to them>"
}
No prose, no markdown, no apologies.
02 · Inbound lead triage n8n + Claude

// n8n nodes

1. Webhook (Trigger)                · POST /webhook/contact-form
2. HTTP → Apollo                    · POST /v1/people/match (email → enrich)
3. HTTP → Anthropic                 · claude-sonnet-4-7 · brief + score
                                       (see prompt below)
4. Switch node (by score)
   ├─ score >= 8 → Slack DM         · ping KAM on duty, full brief
   ├─ score 4–7  → HubSpot          · add to nurture sequence
   └─ score < 4   → HubSpot         · archive with reason
5. HubSpot                          · create deal, attach AI brief as note

03 · Half-yearly review prep n8n + Claude

// n8n nodes

1. Cron                             · check daily for calendar events tagged "review"
2. Google Calendar (or Outlook)     · find events at T-7 days
3. HTTP → HubSpot                   · pull last 6 months: deals, tickets,
                                       activity, NPS, revenue
4. Gmail API (or Outlook)           · pull e-mail thread digest with customer
                                       (subject + body excerpt + dates)
5. HTTP → Anthropic                 · claude-sonnet-4-7 · snapshot (prompt below)
6. Notion API                       · upsert page into account database
7. Slack DM → KAM                   · link + 60-sec summary

// claude · review snapshot prompt

You write half-yearly customer review briefs for a B2B sales lead.
Be specific, no marketing language, no hedging.

Customer: {{ account.name }}
Period: last 6 months vs. prior 6 months

Data:
- HubSpot snapshot: { revenue, deals_closed, tickets_open,
                      escalations_in_period, nps_movement }
- E-mail digest: {{ thread_summary }}

Output a structured one-page brief:
1. Revenue health (warning signs, unresolved issues, sentiment)
2. Growth conversations (parts of their org we don't serve yet,
   budgets we haven't touched, things they wished we offered)
3. What's new on our side they should know about
4. Three suggested talk-tracks for the meeting,
   ordered by importance

Max 350 words. End with one specific commitment we should ask for.
04 · Renewal radar n8n + Claude

// n8n nodes

1. Cron                             · daily 06:00
2. HTTP → HubSpot                   · query contracts expiring in 180 / 120 / 60 days
3. Loop over each account
4. HTTP → HubSpot                   · pull engagement: last 90 days activity,
                                       open tickets, NPS, expansion notes
5. HTTP → Anthropic                 · risk-score per account (prompt below)
6. Switch node
   ├─ at-risk    → Slack #renewals (alert + recovery action)
   ├─ on-track   → Calendar pre-book renewal conversation
   └─ expansion  → HubSpot deal create with note
7. HubSpot                          · log next-action with owner + date

// claude · renewal risk prompt

You evaluate B2B account renewal risk for a sales lead.

Account: {{ account.name }} · ACV €{{ acv }} · renewal in {{ days_to_renewal }} days

Signals:
- Last 90d activity: {{ activity }}
- Open tickets: {{ tickets }}
- NPS trend: {{ nps_trend }}
- Last review meeting: {{ last_review_date }}
- Recent escalations: {{ escalations }}

Return JSON only:
{
  "classification": "at-risk" | "on-track" | "expansion-opportunity",
  "confidence": <0.0-1.0>,
  "reason": "<one paragraph why>",
  "recommended_action": "<the specific next move>",
  "owner": "KAM" | "CSM" | "exec"
}
05 · Crisis escalation routing n8n + Claude

// n8n nodes

1. Inbox watcher (Gmail / Outlook)  · sentiment trigger or keyword match
                                       (delay, complaint, missing, unacceptable, …)
2. HTTP → Anthropic                 · triage + draft (see prompts below)
3. Switch on category
   ├─ ops      → Slack #ops-escalations
   ├─ billing  → Slack #billing
   ├─ product  → Slack #product
   └─ sla      → Slack #sla-watch
4. Always: tag KAM on the thread
5. HubSpot                          · create ticket, attach AI draft
6. Gmail draft                      · save Claude's acknowledgement as draft
7. Slack DM → KAM                   · "draft is in your inbox, read and ship"

// claude · triage + acknowledgement prompt

You handle an angry customer e-mail. Two tasks:

(A) Triage. Return JSON:
{
  "category": "ops" | "billing" | "product" | "sla",
  "severity": 1 | 2 | 3,
  "expectation": "<what the customer expected>",
  "damage": "<what the failure cost them>"
}

(B) Draft a reply for the KAM to send. Rules:
- Acknowledge first. Don't explain.
- Name the expectation + damage clearly.
- One specific recovery action with a time.
- Offer one toolkit piece (credit / expedite / replacement) — only one.
- DO NOT promise "this won't happen again".
- DO NOT discuss root cause in this reply.
- Max 120 words. Plain text. No marketing language.

Customer e-mail:
"""
{{ email_body }}
"""

                                       (prompt below)
5. Filter                           · score >= 7
6. HTTP → Apollo                    · find hiring decision-maker

7. HTTP → Anthropic                 · draft cover letter + Easy-Apply answers
                                       + LinkedIn connect note
8. Notion API                       · upsert into "active applications" database
9. Slack DM → me                    · digest with all drafts, one tap to send each
10. [Manual] Apply on the platform (Easy Apply auto-fill or copy-paste)
11. [Manual] Send LinkedIn connect request to decision-maker
12. Webhook (LinkedIn connect accepted, via PhantomBuster polling)
13. HTTP → Anthropic                · draft a thank-you / intro DM
14. Slack DM → me                   · "ready to send, here it is" — I review & ship

// claude · cover letter + outreach prompt

You write the first draft of a job application for Onur Berrak.

CONTEXT:
- 11 years B2B sales, last 3.5 at aveato (Berlin's largest B2B caterer)
- €3.58M closed across 2023–2025; Top 2–3 of the sales floor 3 yrs running
- AI-native operator: Claude / n8n / HubSpot / AWS ML+AI Fundamentals
- Trilingual: TR native, DE C1 (DSD II), EN B1

ROLE:
{ company, title, description, hiring_manager_name }

Produce three drafts:

1. COVER LETTER (max 200 words):
   - Open with what they're solving for (from the JD).
   - One concrete sales outcome that maps to their need.
   - Why now: I want sales-with-AI roles, not pure sales.
   - Close with a specific next step (15 min call).
   - No marketing clichés. No "passionate". No "synergy".

2. LINKEDIN CONNECT NOTE (max 300 chars):
   - Reference one specific thing about them or the company.
   - Hint at the application but don't sell it.
   - Sign off naturally.

3. POST-CONNECT MESSAGE (max 600 chars, sent only AFTER they accept):
   - Thank them.
   - One paragraph why I applied.
   - Ask the smallest possible question (e.g. "is hiring still active?").
   - Don't pitch.

Return as JSON with keys: cover_letter, connect_note, post_connect.
